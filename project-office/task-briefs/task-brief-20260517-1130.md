### 5. Edge Cases

#### GET /api/v1/experience/rita/backtest-daily
1. No completed backtest runs → return [] without exception
2. Active instrument has no matching runs → return [] gracefully
3. Backtest daily table empty for the latest run → return []
4. Database connectivity failure → catch and return []

#### GET /api/v1/experience/rita/risk-timeline
1. instrument param casing mismatch → preserve .upper() normalisation from existing route
2. phase param other than "all" → preserve phase=_ forward-compat stub
3. Single data point → rolling_vol and var_95 return None, no StatisticsError
4. All portfolio_value entries None → drawdown calc must not divide by zero (default to 1.0)

#### GET /api/v1/experience/rita/training-history
1. No runs for requested instrument → return []
2. backtest_mdd is None → bt_constraints None guard must be preserved
3. Non-chronological ordering → sort by recorded_at then reverse for newest-first

#### POST /api/v1/portfolio/adjust-position-action
1. timestamp not in JS payload → inject datetime.now(timezone.utc) server-side (else 422)
2. date sent as empty string → handle gracefully or accept str type with default
3. Duplicate action submission → no dedup needed, each call gets new manoeuvre_id
4. ManoeuvreService.record() raises → catch and return 500 with logged error
