## [Architect] Design

### Feature Summary

Phase 2 migrates the `rita/` and `ops/` JavaScript modules away from their local copies of API, utility, and chart helpers to import from the canonical `shared/` layer created in Phase 1. The result is that `shared/api.js`, `shared/utils.js`, `shared/charts.js`, and `shared/nav-base.js` become the single source of truth for all three dashboards, and the local `rita/` and `ops/` helpers either become thin re-export wrappers or are deleted.

### API Contract

No new API endpoint — pure JS refactoring task.

**Shared module export inventory:**

**`dashboard/js/shared/api.js`**
- `export const apiBase` — `() => string`
- `export async function api(path, method?, body?)` — throws on non-OK response
- `export async function apiFetch(url, options?)` — returns null on error; adds X-Request-ID header

**`dashboard/js/shared/utils.js`**
- `export const fmt` — `(v, d=2) => string`
- `export const fmtPct` — `(v) => string`
- `export const fmtMs` — `(v) => string`
- `export function setEl(id, html)`
- `export function appendResult(containerId, html)`
- `export function badge(status)` — single-argument

**`dashboard/js/shared/charts.js`**
- `export function destroyChart(id)`
- `export function mkChart(id, config)`
- `export const C` — color palette
- `export function chartOpts(label, tickCb, labels)`

**`dashboard/js/shared/nav-base.js`**
- `export function createNavRegistry()`

**Critical divergence:** `ops/utils.js badge(text, cls)` uses TWO arguments (incompatible with shared `badge(status)`). The merged `ops/utils.js` must keep `badge` as a local two-argument definition.

**Critical divergence:** `shared/api.js api()` THROWS on error; old `ops/api.js api()` returned null. The ops callers that use `api()` wrap in try/catch — they are safe. `daily-ops.js` call sites must be audited.
