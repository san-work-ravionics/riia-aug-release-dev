## [Architect] Design

**Feature summary:** This feature makes the RITA instrument universe self-service: an operator can type a company name on the Daily Ops page, pick from Yahoo Finance search results, and have the instrument's OHLCV data fetched, normalized, persisted to the database, and made available for signals, training, and the geography overview — without developer intervention. Run A delivers the backend pipeline (data fetch, normalization, DB registration, cache seeding) and the two API endpoints that the frontend (Run B) will call.

**API contract — Endpoint 1: GET /api/v1/instrument/search**
| Field | Value |
|---|---|
| Method | GET |
| Path | `/api/v1/instrument/search` |
| Query params | `q: str` (required, min length 2) |
| Request body | none |
| Response shape | `list[dict]` — ticker (str), name (str), exchange (str), currency (str), country (str), quote_type (str — always "EQUITY"). Max 10 results. |
| Auth required | no |
| Errors | 400 if q < 2 chars; 502 if yfinance unreachable |

**API contract — Endpoint 2: POST /api/v1/instrument/onboard**
| Field | Value |
|---|---|
| Method | POST |
| Path | `/api/v1/instrument/onboard` |
| Query params | none |
| Request body | ticker (str, required), name (str, required), exchange (str, required), currency (str, required), country_code (str, required), lot_size (int, optional) |
| Response shape | status ("ok"), ticker (str), rows_fetched (int), rows_seeded (int), raw_path (str), input_path (str) |
| Auth required | no |
| Errors | 409 duplicate ticker; 400 bad ticker or < 100 rows; 502 yfinance failure |

**Pipeline execution order (POST /api/v1/instrument/onboard):**
1. Check instruments table → 409 if found
2. `fetch_raw_data(ticker)`
3. `process_to_input(ticker, raw_path)`
4. `InstrumentRepository.upsert` with `is_available=True`, currency, lot_size
5. `seed_market_cache(db, ticker, currency)`

**Geography Overview Change:**
`GET /api/v1/experience/rita/geography-overview` — path and auth unchanged. Replace hardcoded `_GEO_REGIONS` list with dynamic query: `InstrumentRepository(db).read_all()` filtered to `is_available=True`, map `country_code` → region bucket (IN→India; US→US; NL/DE/FR/GB/BE/CH and other EU codes→EU; else→Other). `GeographyOverviewResponse` schema unchanged.
