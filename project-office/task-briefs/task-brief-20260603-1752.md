## [Architect] Design

**Feature summary:** Phase 2 wires the Portfolio Hedge wizard's JS module to the persistence endpoints delivered in Phase 1: on page load, the saved hedge plan (hedged_ids, coverage, scenario_tab) is fetched via GET and silently restores the user's previous wizard state, so the user sees their last selections rather than defaults every visit. On every user change (checkbox toggle or coverage slider), the plan is persisted via debounced PUT (400 ms quiet period), so state is saved without requiring an explicit "Save" action.

**API contract — GET:**
| Field | Value |
|---|---|
| Method | GET |
| Path | `/api/v1/experience/fno/hedge-plan` |
| Query params | none |
| Request body | none |
| Response shape | key_id: str, hedged_ids: list[str], coverage: int, scenario_tab: str, duration: str (always "1y"), updated_at: datetime |
| Auth required | Yes — JWT Bearer from `sessionStorage.getItem('auth_token')` |
| 404 meaning | No portfolio key or no saved plan — JS must use current `_state` defaults silently |
| 401 meaning | Token expired — existing redirect-to-auth pattern applies |

**API contract — PUT:**
| Field | Value |
|---|---|
| Method | PUT |
| Path | `/api/v1/experience/fno/hedge-plan` |
| Query params | none |
| Request body | hedged_ids: list[str], coverage: int (0–100), scenario_tab: str |
| Response shape | key_id: str, hedged_ids: list[str], coverage: int, scenario_tab: str, duration: str, updated_at: datetime |
| Auth required | Yes — JWT Bearer from `sessionStorage.getItem('auth_token')` |
| Error handling | 4xx/5xx → JS logs `console.warn`, does NOT crash wizard |

> Both endpoints are pre-existing (Phase 1). Engineer does NOT create new backend code.

**Frontend target:**
| Item | Value |
|---|---|
| JS module | `dashboard/js/fno/portfolio-hedge.js` — MODIFY existing, do not create new |
| Section id | `sec-portfolio-hedge` — existing, no HTML changes required |
| New function: loadHedgePlan(token) | Internal async, called in `loadPortfolioHedge()` after defaults set, before render. On 404 → silently uses defaults. On 401 → `api()` redirects. On other errors → `console.warn`. |
| New function: saveHedgePlan() | Internal async, builds PUT body from `_state`: `{ hedged_ids: [..._state.hedgeChecked], coverage: _state.coverage, scenario_tab: _scenarioTab }`. On failure → `console.warn` only. |
| New function: _debouncedSave() | Clears `_savePlanTimer`, sets 400ms `setTimeout` → `saveHedgePlan()`. Called from `phToggleHedge()`, `phSetCoverage()`, `phSetScenarioTab()`. |
| Module-level constants | `let _savePlanTimer = null;` and `const _SAVE_DEBOUNCE_MS = 400;` |
| Window bindings | None — all three functions are internal |

**Edge cases to handle:**
1. GET 404 → silently use defaults, no error shown
2. GET 401 → `api()` redirects to auth automatically
3. GET 500/network → `console.warn`, defaults remain, wizard still renders
4. PUT failure → `console.warn` only, wizard does not crash
5. Rapid toggles (< 400ms) → debounce fires only once on final state
6. `hedged_ids` empty list → valid, PUT accepts `[]`
7. `coverage=0` → valid, `HedgePlanCreate` accepts 0
8. `hedged_ids` restored with IDs not in current portfolio → ignored silently (no matching checkbox rendered)
9. `scenario_tab` unrecognised → `_scenarioTab` set to returned value, `_highlightScenarioTab` skips silently
10. JS field type discipline → `hedged_ids` used as strings (no `parseInt`), `coverage` assigned as int
