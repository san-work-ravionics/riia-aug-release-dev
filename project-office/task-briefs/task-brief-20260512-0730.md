#### backfill_metrics.py (new)

- Input: all `run-*.json` files in `riia-ai-org/agent-ops/runs/`
- For each run missing the new fields, adds without overwriting:

| Field | Backfill value |
|---|---|
| `retry_count` | `0` |
| `abandoned` | `false` |
| `loop_events` | `0` |
| `hitl_events` | `[]` |
| `human_score` | `{"accuracy": null, "relevance": null, "planning_ok": null, "csat": null, "time_saved_hours": null}` |
| `token_forecast` | Computed retrospectively: complexity derived from `total_tokens_estimated` (<30K ‚Üí small, 30-60K ‚Üí medium, >60K ‚Üí large). Per-role split proportional to historical averages (PM 7612, Arch 9975, Eng 31112, QA 11300, TW 6650). `forecast_error_pct` = 0 for backfilled runs. `basis_runs` = count of prior runs for same `feature_type`. |
| Engineer `grounding_checks.memory_used` | `false` if absent |
| Engineer `grounding_checks.tool_error_handled` | `false` if absent |

- Output: overwrites each run JSON in place (`json.dump(..., indent=2)`)
- Invocation: `python riia-ai-org/agent-ops/backfill_metrics.py`
- Must never touch any file under `rita_input/`

#### aggregate_metrics.py ‚Äî 7 new compute sections

1. `task_completion` ‚Äî `tsr`, `first_attempt_success_rate`, `partial_completion_rate`, `abandonment_rate`
2. `quality` ‚Äî `avg_accuracy_score`, `avg_relevance_score`, `avg_csat`, `planning_accuracy_rate`, `grounding_pass_rate`
3. `token_forecasting` ‚Äî `avg_forecast_error_pct`, `by_complexity`, `by_feature_type`
4. `efficiency` ‚Äî `avg_duration_minutes`, `avg_tokens_per_run`, `avg_retry_count`, `avg_time_saved_hours`
5. `reliability` ‚Äî `error_recovery_rate`, `graceful_degradation_rate`, `loop_event_total`
6. `hitl` ‚Äî `escalation_rate`, `avg_corrections_per_run`, `total_hitl_events`
7. `agentic` ‚Äî `context_adherence_rate`, `memory_utilization_rate`, `loop_detection_rate`

`forecast_error_pct` per run = `abs(total_tokens_estimated - token_forecast.total_forecast) / token_forecast.total_forecast √ó 100`

**4 threshold alert conditions** (stdout after writing metrics.json):

| Condition | Stdout message |
|---|---|
| Any FC code total > 3 | `[ALERT] FC-{code} has fired {N} times ‚Äî review skill file rule` |
| Any role first_pass_rate < 0.70 | `[ALERT] {role} first-pass rate {N}% ‚Äî grounding checks need review` |
| `quality.avg_csat < 3.5` (non-null) | `[ALERT] CSAT {N}/5 below threshold ‚Äî review last 3 runs` |
| `token_forecasting.avg_forecast_error_pct > 35` | `[ALERT] Token forecast off by {N}% on average ‚Äî recalibrate multipliers` |

`skill_version_history` entries gain 3 new optional fields: `improvement_applied`, `before_first_pass_rate`, `after_first_pass_rate`.

### API Contract

| Field | Value |
|---|---|
| Method | GET |
| Path | `/api/experience/ops/token-forecast` |
| Auth required | Yes ‚Äî JWT |

**Query parameters:**

| Name | Type | Required |
|---|---|---|
| `feature_type` | string | required ‚Äî one of: `rita`, `ops`, `fno`, `invest-game` |
| `files_to_change` | string | required ‚Äî one of: `small`, `medium`, `large` |
| `new_endpoint_or_model` | string | required ‚Äî one of: `none`, `one`, `both` |
| `frontend_scope` | string | required ‚Äî one of: `none`, `panel`, `page` |
| `integration_type` | string | required ‚Äî one of: `additive`, `extends`, `cross-cutting` |

**Response shape** (`TokenForecastResponse`):

| Field | Type |
|---|---|
| `complexity` | string |
| `complexity_score` | float |
| `feature_type` | string |
| `per_role` | dict[str, int] ‚Äî keys: pm, architect, engineer, qa, techwriter |
| `total_forecast` | int |
| `confidence` | string |
| `basis_runs` | int |

- Pydantic schema file: `src/rita/schemas/token_forecast.py`
- Class name: `TokenForecastResponse`
- Endpoint reads `metrics.json` at call time ‚Äî no DB dependency. Path resolved at call time, never at module level.
- Fallback: if `basis_runs < 1` for the requested `feature_type`, uses global per-role averages with `confidence: "¬±40%"` and `basis_runs: 0`.
- Error: if `metrics.json` unavailable, returns HTTP 503 `{"detail": "metrics.json unavailable"}`.

1. `runs/` has no JSON files ‚Äî `backfill_metrics.py` exits cleanly with message; `aggregate_metrics.py` writes empty-but-valid `metrics.json`
2. Run log missing `token_forecast` block ‚Äî JS guards: `r.token_forecast?.total_forecast ?? 0`; `aggregate_metrics.py` skips null/zero values in `forecast_error_pct`
