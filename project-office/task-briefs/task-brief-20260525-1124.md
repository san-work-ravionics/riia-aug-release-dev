## [Architect] Design

**Feature summary:** The Strategy Comparison card adds a fifth card to the RITA Learnings section that runs five rule-based trading strategies (Buy and Hold, Value Investing, Momentum Investing, Swing Trading, Support-Resistance 52-Week H/L) against any instrument's OHLCV data for a selected year (2024 or 2025). It renders a 7-panel Chart.js performance dashboard together with a summary metrics table and auto-fired AI commentary, giving users a side-by-side view of how each strategy would have performed on any available instrument.

**API contract — GET /api/v1/experience/rita/strategy-comparison:**
| Field | Value |
|---|---|
| Method | GET |
| Path | `/api/v1/experience/rita/strategy-comparison` |
| Query params | `instrument: str` (default: active instrument), `year: int` (default: 2025, accepted: 2024, 2025) |
| Request body | none |
| Auth required | no |

Response shape — `StrategyComparisonResponse`:
| Field | Type |
|---|---|
| `instrument` | `str` |
| `year` | `int` |
| `dates` | `list[str]` — ISO date strings for x-axis |
| `strategies` | `list[StrategyResult]` — always 5 entries |
| `summary` | `list[StrategySummaryRow]` — tabular metrics |

`StrategyResult` fields: `name: str`, `equity: list[float]`, `color: str`

`StrategySummaryRow` fields: `name: str`, `total_return_pct: float`, `sharpe: float`, `max_drawdown_pct: float`, `n_trades: int`, `win_rate_pct: float`, `final_value: float`

**Commentary endpoint — POST /api/v1/commentary (extended, no new path):**

Extend `_DISPATCH` with `("rita", "strategy-comparison")` key → `_handle_strategy_comparison` handler. `page` field is `str` (not enum) so no schema change. Handler takes `req.instrument`, produces a conceptual narrative comparing the 5 strategy trade-offs.
