## [Architect] Design

**Feature summary:** This feature adds two complementary capabilities to the RITA platform: a client-side session cache module (`api-cache.js`) that eliminates redundant API calls across rita/ops JS modules, and a full-stack API call monitoring system — comprising a FastAPI middleware that logs every request's path, method, latency, and status code to a new `api_call_log` database table, an experience-tier endpoint that aggregates those logs per path+method with p50/p95 percentiles and error counts, a new Ops dashboard "API Metrics" panel displaying those aggregates in a filterable table, and an `aggregate_metrics.py` addition that writes an `api_metrics` summary block to `metrics.json` for the Agent Builds KPI feed.

**API contract — Primary New Endpoint: GET /api/experience/ops/api-metrics**
| Field | Value |
|---|---|
| Method | GET |
| Path | `/api/experience/ops/api-metrics` |
| Query params | `limit: int = Query(default=200)`, `method: Optional[str] = Query(default=None)`, `path_prefix: Optional[str] = Query(default=None)` |
| Request body | none |
| Response shape | `items: list[ApiMetricsRow]` — each row: `path: str`, `method: str`, `call_count: int`, `p50_ms: Optional[float]`, `p95_ms: Optional[float]`, `error_count: int`, `error_rate_pct: float`, `last_called_at: Optional[str]` |
| Auth required | no |

Response root object: `ApiMetricsResponse(items: list[ApiMetricsRow])`.

**Middleware:** `ApiCallLogMiddleware` in `middleware.py`, registered in `main.py` via `app.add_middleware()`. Fires after `call_next`, captures `path`, `method`, `status_code`, `duration_ms`, `called_at`. Skips `/health`, `/readyz`, `/metrics`, `/docs`, `/openapi.json`, `/dashboard` prefix. Writes one row per request using `SessionLocal()` opened and closed in `try/finally`. On DB write failure: `log.warning(...)` and continue — never raise.

**aggregate_metrics.py:** `compute_api_metrics(db_path)` function reads `api_call_log` via `sqlite3` direct connection. Returns `total_calls`, `unique_endpoints`, `overall_error_rate_pct`, `top_5_by_calls`, `generated_at`. Written to `metrics.json` under key `"api_metrics"`. Alert if `overall_error_rate_pct > 5.0`.
