### Phase 0 — Overview Page (Geography Panels)

**Phase summary:**
The existing RITA Overview / home section gains three side-by-side geography panels (US, EU, India) each displaying four tracked instruments with their current price, daily return, and a live sentiment badge. Users get an at-a-glance view of all monitored markets before diving into instrument-specific analysis.

**Nav / page changes:**

- New DOM: `div#geo-panels` placeholder inside the current market/overview section of `rita.html`.

**Files to touch:**
| File | Change |
|---|---|
| `riia-jun-release/dashboard/rita.html` | Add `<div id="geo-panels"></div>` sub-block inside `sec-market` |
| `riia-jun-release/dashboard/js/rita/market-signals.js` | Add `loadGeoPanels()` function; call from `loadMarketSignals()` |
| `riia-jun-release/dashboard/js/rita/main.js` | Expose `window.loadGeoPanels`; add to `refresh()` call list |
| `riia-jun-release/src/rita/api/experience/rita.py` | Add `GET /api/experience/rita/geography-overview` endpoint |
| `riia-jun-release/src/rita/schemas/geography.py` | New schemas: `GeoInstrument`, `GeoRegion`, `GeographyOverviewResponse` |

**API contract:**
**GET** `/api/experience/rita/geography-overview`
```json
{
  "regions": [
    { "region": "US", "instruments": [ { "id", "name", "flag", "close", "daily_return_pct", "signal" }, "..." ] },
    { "region": "EU", "instruments": ["..."] },
    { "region": "India", "instruments": ["..."] }
  ]
}
```
Auth: no JWT required.

**Edge cases:**
1. Instrument has no market data — card shows `—` for price, `neutral` signal badge; panel still renders.
2. Fewer than 4 instruments for a geography — renders however many exist without crashing.
3. `daily_return_pct` is null — display `—` with neutral badge.
4. All regions empty — show "No geography data configured" message.

**Implementation order:**
1. Pydantic schemas (`geography.py`)
2. Experience endpoint in `rita.py`
3. `div#geo-panels` placeholder in `rita.html`
4. `loadGeoPanels()` in `market-signals.js`
5. Wire into `main.js`
