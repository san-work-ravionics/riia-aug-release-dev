## [Architect] Design

**Feature summary:** Add a read-only `ms-last-updated` label to the market signals panel header that displays the date of the most recent data row returned by the existing `/api/v1/market-signals` endpoint. This is a pure frontend change — no new API contract, no new JS module, and no new backend code.

**API contract:**
| Field | Value |
|---|---|
| Method | GET — existing endpoint, no new endpoint needed |
| Path | `/api/v1/market-signals` (existing) |
| Query params | timeframe, periods, instrument (existing) |
| Request body | none |
| Response shape | `date` (string, last row used for timestamp) — existing field |
| Auth required | no |

**Frontend target:**
| Item | Value |
|---|---|
| JS module | `dashboard/js/rita/market-signals.js` (modify existing) |
| Section id | `sec-market-signals` (existing) |
| DOM element IDs | `ms-last-updated` (new span in panel header) |
| Window bindings | none (no new onclick handlers) |

**Files to touch:**
| File | Change |
|---|---|
| `dashboard/js/rita/market-signals.js` | Read `last.date`, format it, call `setEl('ms-last-updated', ...)` after successful data load; call `setEl('ms-last-updated', '—')` in empty-data guard and catch block |
| `dashboard/rita.html` | Add `<span id="ms-last-updated">` in `.hdr-actions` div of market signals panel header |
| `project-office/specs/Spec_JS_Code.md` | Add `ms-last-updated` to DOM targets table for market-signals section |

**Edge cases to handle:**
- Empty API response (`rows` null or zero-length): set `ms-last-updated` to `—` in the existing early-return guard
- API/network error (exception thrown): set `ms-last-updated` to `—` in the existing catch block
- `last.date` null even when rows present: guard with `last.date ? format(...) : '—'`
- Stale value across tab switches: `loadMarketSignals()` is called on every `switchMsTab()` so label refreshes automatically

**Timestamp format:** `"Last updated: 28 Apr 2026"` — `new Date(last.date + 'T00:00:00').toLocaleDateString('en-GB', { day: 'numeric', month: 'short', year: 'numeric' })`. The `T00:00:00` suffix forces local-time parsing to avoid UTC off-by-one. Matches the `en-GB` locale pattern already used in `trades.js`.
