- **Companion `_yf.csv` date-format mixing (medium):** The existing NIFTY `merged.csv` and BANKNIFTY `banknifty_daily_25yr_rounded.csv` use IST-format timestamps (`1999-07-01 00:00:00+05:30`). The spec correctly isolates new yfinance rows into separate `_yf.csv` companion files to avoid misparsing. The `load_instrument_data()` extension must preserve the existing `load_ohlcv_csv()` IST-detection path and only merge after each file is independently parsed. Any regression here would silently corrupt NIFTY/BANKNIFTY data used by the RL model.
- **Targeted upsert — no deletes (medium):** `upsert_cache_delta` must strictly insert new `(instrument, date)` pairs only, with no `DELETE` or truncation. A prior incident (Day 16 DB migration) showed that accidental full-table operations are destructive and hard to recover. Engineer must use `db.add_all()` with an existence check, not `merge()` or bulk replace.
- **`yf_ticker` Alembic migration (low-medium):** Adding a nullable `VARCHAR` column to `instruments` is a safe DDL operation, but the migration must be compatible with the existing SQLite engine and must not break any ORM model that already maps the `instruments` table. The migration must be verified with `alembic upgrade head` + `alembic downgrade base` in CI before merge.
- **yfinance network dependency (low):** yfinance is an external HTTP dependency with occasional 502/429 responses. The spec calls for per-instrument error catching so one failure does not abort the rest. This is correctly scoped; the slash command and standalone script must handle timeouts gracefully.
- **`nifty_manual.csv` / `banknifty_manual.csv` data preservation (low):** These files contain historical appended rows that may not yet be in the DB cache. Before superseding the manual workflow, the Engineer must confirm that all dates already present in those files are either already in `market_data_cache` or are covered by the first `refresh_all` run. The files must be retained (not deleted) — the spec correctly states "files retained but not required."

### 5. Edge Cases

1. **EC-1: No gap (gap_days == 0)** — check_gap returns 0, skip yfinance call, result is {status: "current"}
2. **EC-2: yfinance 502/429/timeout** — per-instrument try/except, loop continues, {status: "error", error: str(exc)}
3. **EC-3: Duplicate (instrument, date) in market_data_cache** — upsert_cache_delta queries existing dates first, only inserts missing ones via db.add_all(); no db.merge(), no DELETE
4. **EC-4: NIFTY/BANKNIFTY companion file date-format conflict** — both files independently loaded via load_ohlcv_csv() before concat; dedup uses keep="last" so yfinance data wins on overlap
5. **EC-5: yfinance MultiIndex DataFrame (yfinance >= 0.2.x)** — flatten columns with df.columns = [col[0] for col in df.columns] before CSV write (same pattern as instrument_onboard.py)
6. **EC-6: Migration committed but not applied** — hard DoD gate: `alembic upgrade head` must show "Running upgrade" output; otherwise app crashes with OperationalError on first refresh call
