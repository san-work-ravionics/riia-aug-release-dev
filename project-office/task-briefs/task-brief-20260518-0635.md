## [Architect] Design

**Feature summary:** This feature adds an Instrument Onboarding sub-panel to the existing Daily Ops section in the Ops dashboard. Ops users can search Yahoo Finance by keyword, select a result, and trigger the full onboarding pipeline (data fetch, normalization, DB registration, market cache seed) — all from the ops.html UI without leaving the dashboard.

**Tier decision:** No Experience tier wrapper required. The two Run A endpoints (`/api/v1/instrument/search` and `/api/v1/instrument/onboard`) are called directly from JS using the existing `api()` helper — consistent with how `daily-ops.js` already calls other workflow endpoints.

**API contract — Endpoint 1: Search**
| Field | Value |
|---|---|
| Method | GET |
| Path | `/api/v1/instrument/search` |
| Query params | `q: string` — minimum 2 characters |
| Request body | none |
| Response shape | `ticker: string`, `name: string`, `exchange: string`, `currency: string`, `country: string`, `quote_type: string` (per item) |
| Auth required | no |

**API contract — Endpoint 2: Onboard**
| Field | Value |
|---|---|
| Method | POST |
| Path | `/api/v1/instrument/onboard` |
| Query params | none |
| Request body | `ticker: string`, `name: string`, `exchange: string`, `currency: string`, `country_code: string`, `lot_size: int \| null` |
| Response shape | `status: string`, `ticker: string`, `rows_fetched: int`, `rows_seeded: int`, `raw_path: string`, `input_path: string` |
| Auth required | no |

Error codes to handle: 400 (bad ticker), 409 (duplicate), 422 (missing fields), 502 (yfinance unreachable)
