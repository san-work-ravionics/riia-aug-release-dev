## [Architect] Design

### Feature Summary

Phase 2 exposes the User Portfolio Store — built in Phase 1 — through the API layer. It delivers two workflow-tier endpoints (POST and GET at `/api/v1/user-portfolio`) for authenticated save/retrieve operations, one experience-tier endpoint (GET at `/api/v1/experience/user-portfolio`) for authenticated read-only consumption by the frontend, and an update to the Google OAuth login flow so that a `state` query parameter can route the post-login redirect to either `/dashboard/index.html` (RITA) or `/dashboard/fno.html` (FnO), unblocking Phase 4's FnO auth gate.

### API Contract

**Endpoint 1 — POST /api/v1/user-portfolio (workflow tier)**

| Field | Value |
|---|---|
| Method | POST |
| Path | `/api/v1/user-portfolio` |
| Query params | none |
| Request body | `name: str \| None` (optional, defaults to "My Portfolio"); `holdings: list[HoldingItem]` where `HoldingItem` is `{ instrument_id: str, allocation_pct: float (>0) }` |
| Response shape | `portfolio_id: str`, `key_id: str`, `name: str`, `holdings: list[HoldingItem]`, `created_at: datetime`, `updated_at: datetime`, `is_active: bool` |
| Status codes | 201 Created; 401 no JWT; 422 empty holdings or sum != 100 |
| Auth required | Yes — JWT via `get_current_user` |

**Endpoint 2 — GET /api/v1/user-portfolio (workflow tier)**

| Field | Value |
|---|---|
| Method | GET |
| Path | `/api/v1/user-portfolio` |
| Query params | none |
| Request body | none |
| Response shape | `UserPortfolioOut` (same schema) |
| Status codes | 200; 401 no JWT; 404 no active portfolio |
| Auth required | Yes — JWT |

**Endpoint 3 — GET /api/v1/experience/user-portfolio (experience tier)**

| Field | Value |
|---|---|
| Method | GET |
| Path | `/api/v1/experience/user-portfolio` |
| Query params | none |
| Request body | none |
| Response shape | `UserPortfolioOut` (same schema — reuse Phase 1 schema) |
| Status codes | 200; 401 no JWT; 404 no active portfolio |
| Auth required | Yes — JWT via per-route `Depends(get_current_user)` |
| Tier rule | Read-only — no db.commit(), no mutations. Call UserPortfolioRepo directly (no service layer). |

**Auth Update — state param on GET /auth/google/login + callback redirect**

- `GET /auth/google/login` accepts optional `state: str | None = None` query param. Append `&state={state}` to Google OAuth URL when not None.
- `GET /auth/google/callback` accepts `state: str | None = None`. Maps: `state == "fno"` → `/dashboard/fno.html?token=…`; anything else → `/dashboard/index.html?token=…` (current behaviour).
- redirect_uri sent to Google stays `/auth/google/callback` — state is a separate payload.

1. Unauthenticated request → 401 (all 3 endpoints)
2. Holdings sum != 100.0 → 422 (caught from ValueError in service)

5. Invalid/unrecognised state param → silently defaults to RITA redirect (backward-compatible)
6. Experience-tier write prohibition — no db.commit() reachable in experience router
