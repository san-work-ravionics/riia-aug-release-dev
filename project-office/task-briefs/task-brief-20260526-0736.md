## [Architect] Design

### 1. Feature Summary
This feature adds a purpose-built gateway landing page at `/mobile` that acts as a hub for all RITA platform apps. When a mobile user arrives, they see five app cards clearly labelled as either "Mobile Ready" (RITA Trading, Invest Game) or "Desktop Only" (FnO, Ops, Data Science), with appropriate links and escape-hatch affordances for users who want to force-open a desktop dashboard anyway.

### 2. API Contract
This is a page route, not a data API. No Pydantic schema is needed.

| Attribute | Value |
|---|---|
| Method | GET |
| Path | `/mobile` |
| Handler | `def mobile()` in `src/rita/main.py` |
| Response type | `FileResponse` |
| File served | `mobileapp/gateway.html` |
| Status code | 200 on success |
| Auth required | No |
| `include_in_schema` | `False` |
| Query params consumed | None |

Route placement constraint: `@app.get("/mobile")` must appear BEFORE the `app.mount("/mobileapp", ...)` static mount.

### 3. Frontend Target
File: `riia-jun-release/mobileapp/gateway.html` — CREATE new standalone self-contained HTML page. NOT a JS module. NOT a dashboard section. No script tags. All CSS inline.

5 App cards with DOM IDs:
