## [Architect] Design

### Feature Summary

The Portfolio Builder page in the FnO dashboard lets a user assemble a 5–10 instrument basket by region, signal, and estimated performance — faster than one-at-a-time clicking. It presents the same instrument universe through three synchronized lenses (region buckets with group select-all, a return-vs-risk scatter map, and a sortable table) plus a guided basket flow with goal presets, all culminating in a sticky final basket and a "Build Portfolio" CTA that hands off to the existing Feature 26 save flow.

---

### API Contract

No new endpoints for Phase 1. Uses existing live endpoints:

| Endpoint | Method | Auth | Role |
|---|---|---|---|
| `/api/v1/experience/rita/geography-overview` | GET | none | Primary data source — region buckets, instrument list, signals, close, daily_return_pct |
| `/api/v1/user-portfolio/` | POST | JWT Bearer (`rita_token`) | "Build Portfolio →" CTA — body: `{ name, holdings: [{ instrument_id, allocation_pct }] }` |
| `/api/v1/experience/user-portfolio` | GET | JWT Bearer (`rita_token`) — required; 401 if missing | Pre-fills basket on page load; 404 treated silently as empty basket |

**GET /api/v1/experience/user-portfolio — Load Existing Portfolio (pre-fill basket)**

| Field | Value |
|---|---|
| Method | GET |
| Path | `/api/v1/experience/user-portfolio` |
| Tier | Experience (Tier 1, read-only) |
| Auth | JWT Bearer (`rita_token`) — required; 401 if missing |
| 404 behaviour | Returns HTTP 404 with `{ "detail": "No active portfolio found" }` — JS treats silently as empty basket |

Response model: `UserPortfolioOut` (HTTP 200)

| Field | Type | Notes |
|---|---|---|
| `portfolio_id` | `string` | Unique portfolio UUID |
| `key_id` | `string` | User portfolio key UUID |
| `name` | `string` | Portfolio display name |
| `holdings` | `HoldingItem[]` | Array of held instruments |
| `holdings[].instrument_id` | `string` | Use to pre-fill `_basket` Set |
| `holdings[].allocation_pct` | `float` | Display in basket item row |
| `created_at` | `datetime` | ISO-8601 creation timestamp |
| `updated_at` | `datetime` | ISO-8601 last-update timestamp |
| `is_active` | `boolean` | Always `true` for active portfolio |

**POST /api/v1/user-portfolio/ — Save Portfolio (Build Portfolio CTA)**

| Field | Value |
|---|---|
| Method | POST |
| Path | `/api/v1/user-portfolio/` |
| Tier | Workflow (Tier 2) |
| Auth | JWT Bearer (`rita_token`) — required |
| Status on success | HTTP 201 Created |
| Status on validation error | HTTP 422 |

Response model: `UserPortfolioOut` (same 7-field shape as GET above)

Error handling in pbBuildPortfolio: empty basket → block POST; 401 → redirect to /auth/google/login?state=fno; 422 → show detail in pb-status-msg; 5xx → generic error in pb-status-msg.

**Response fields consumed from geography-overview:**
`geo.regions[].region`, `.flag`, `.instruments[].id`, `.name`, `.flag`, `.close`, `.daily_return_pct`, `.signal`

**Phase 1 client-side derivations (labeled "(est.)" in UI):**
- `_estRisk(inst)` → 1–5 scale from `|daily_return_pct|`
- `_estReturn(inst)` → `daily_return_pct` used as proxy

### Edge Cases to Handle

1. Empty/null geography-overview response → show `pb-empty`; hide bucket/map/table views.
2. No saved portfolio (404 on user-portfolio GET) → silent; start with empty basket.
3. User not authenticated on "Build Portfolio →" → show sign-in prompt or redirect to `/auth/google/login?state=fno`; no POST call.
4. All instruments in a region have `close === null` → render as disabled/greyed; skip in select-all; show "No live data" note in bucket.
5. Geography API error (non-200, timeout) → show `pb-error` + `pb-error-msg`; preserve existing basket state.
6. Scatter map has no plottable data (all `daily_return_pct` null) → show `pb-map-empty`; hide canvas; do not throw Chart.js error.
7. Basket persistence on section re-load → module-level `_basket` Set persists for session; re-render calls `_renderBasket()` to reflect current state, does not reset.
