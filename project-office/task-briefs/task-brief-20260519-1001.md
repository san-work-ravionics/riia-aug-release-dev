## [Architect] Design

**Feature summary:** Add a client-side internationalisation (i18n) system to all five RITA dashboard surfaces and the mobile PWA, supporting English (EN), Dutch (NL), and French (FR) language switching via a persistent pill-selector capsule in each page header. Language preference is stored in localStorage with no backend involvement.

**API contract:**
| Field | Value |
|---|---|
| Method | No API — pure client-side i18n |
| Path | Language selection stored in localStorage('ritaLanguage') |
| Query params | none |
| Request body | none |
| Response shape | none |
| Auth required | no |

**Frontend target:**
| Item | Value |
|---|---|
| JS module | `dashboard/js/shared/i18n.js` (new shared module) |
| Section id | n/a — cross-page |
| DOM element IDs | `.lang-capsule` buttons (EN / NL / FR); active button receives `.active` class |
| Window bindings | `window.setLanguage` (exposed in each page's main.js) |

**Edge cases to handle:**
1. Missing translation key — fallback to en dict, then raw key string; never blank
2. localStorage unavailable — try/catch in `getLanguage()`/`setLanguage()`; default `'en'`
3. `applyTranslations()` before DOM ready — always called inside DOMContentLoaded
4. Language switch while section mid-load — JS loaders call `t()` at render time, picks up current language
5. Mobile PWA offline — locale dicts are inline objects, no network fetch
6. Capsule active-state on page load — init code reads `getLanguage()` and adds `.active` to matching button
