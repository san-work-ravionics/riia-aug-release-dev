## [Architect] Design

### Feature Summary
Phase 1 creates four new canonical shared JavaScript modules under `dashboard/js/shared/` (api.js, utils.js, charts.js, nav-base.js) and replaces `dashboard/js/rita/charts.js` with a re-export shim — establishing a single source of truth for cross-app utilities without modifying any existing import paths or runtime behavior. This is a purely additive structural foundation; no existing module is deleted or re-wired in this phase.

### JS Module Contracts

**shared/api.js** — New file. Exports: `apiBase() → string`, `async api(path, method?, body?) → Promise<any>` (throws on non-2xx), `async apiFetch(url, options?) → Promise<any|null>` (returns null on error, adds X-Request-ID header).

**shared/utils.js** — New file. Exports: `fmt(v, d?)`, `fmtPct(v)`, `fmtMs(v)`, `setEl(id, html)`, `appendResult(containerId, html)`, `badge(status)`. badge() must use `String(status || '')` coercion (EC-5). fmt() returns '—' on null/undefined/empty string.

**shared/charts.js** — Moved from rita/charts.js. Exports: `destroyChart(id)`, `mkChart(id, config)`, `chartOpts(label, tickCb, labels)`, `C` (color palette). Internal import must change from `'./chart-modal.js'` to `'../rita/chart-modal.js'`.

**shared/nav-base.js** — New file. Exports: `createNavRegistry()` returning `{ register(key, fn), load(key), reset(key), loaders }`. load() must silently no-op for unregistered keys.

### Frontend Target
- New shared modules: dashboard/js/shared/api.js, shared/utils.js, shared/charts.js, shared/nav-base.js
- Re-export shim: dashboard/js/rita/charts.js → `export * from '../shared/charts.js';`
- No new HTML section, no new window bindings
