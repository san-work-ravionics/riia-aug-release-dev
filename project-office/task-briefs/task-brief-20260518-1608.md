### 5. Edge Cases

1. **window.show timing** — all `window.*` bindings must be at top-level module scope in main.js, not inside DOMContentLoaded. ES modules are deferred so the HTML nav onclick attributes are registered before the module runs; any click before load would throw ReferenceError if window.show is not set at module scope.
2. **ds-specific chart modal vs shared chart modal** — `openChartModal`/`closeChartModal` in ds.html are a simpler image-based modal (`<div id="chart-modal">`), NOT the same as `shared/charts.js`'s `openChartModal`. Must live in `ds/utils.js`. Do NOT import from shared/charts.js for these.
3. **`_activeInst` cross-section state** — pipeline.js writes, performance.js / trades.js / experiment-results.js read. Must use `ds/state.js` shared state object. Modules import `{ state }` from `'./state.js'`.
4. **`fmtPctRaw(v, dec)` vs `fmtPct(v)`** — shared fmtPct hardcodes 2 decimal places. Any section that needs a specific decimal count must use `fmtPctRaw` from `ds/utils.js`. Silent substitution will change rendered output (FC-004 risk).
5. **mkTbl(rows, cols)** — no shared equivalent; must live in ds/utils.js and be imported by any section that calls it.
6. **CDN scripts** — lines 10–11 in ds.html (Chart.js + annotation plugin) must NOT be removed. Module is deferred so CDN globals are available before module executes.
7. **Nav-collapse IIFE** — must remain as a plain `<script>` (not type="module"). Converting to module changes execution timing and breaks sidebar toggle.
8. **`show('dashboard', n)` inside pipeline.js** — pipeline.js calls `show()` on completion. Must call `window.show(...)` (not import show) to avoid circular dependency (nav needs loaders dict which is in main.js).
9. **C color palette extension** — `shared/charts.js`'s `C` lacks `ds`, `dsBg`, `t2`, `grid`, `buildBg`, `runBg`, `monBg`, `warnBg`, `dangerBg`. Must define `DS_C = { ...C, ds: '#0E7490', dsBg: 'rgba(14,116,144,.12)', ... }` in `ds/utils.js`.
10. **init() orchestrator** — main.js DOMContentLoaded must replicate the inline `init()`: set default dates, call `loadInstruments()`, `checkStatus()`, `setInterval(checkStatus, 30000)`, `loadActiveInstrument()`, then `loadUnderstand()` as default section.
