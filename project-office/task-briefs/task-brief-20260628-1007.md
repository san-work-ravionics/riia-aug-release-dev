## [Architect] Design

**Feature summary:** Replace the profit-shaped per-step reward in `RIIATradingEnvV2` with a Differential Sharpe Ratio (DSR) reward that directly optimises the graded project objective (Sharpe > 1.0, MDD > -10%), fix the train/eval return-alignment skew, anchor the hard MDD constraint at -10% for all tolerance levels, wire the three-way temporal split into ml_dispatch for honest held-out evaluation, and remove the patch-stack of hand-tuned penalty terms that existed only to bend a return-maximiser toward risk-adjustment.

**API/UI surface:** none — core change only.

**Approach & algorithm:**

*3.5.1 (F2) Causal alignment:* In `step()`, read return from `self.df.iloc[self._start_idx + self._step_idx + 1]` (next bar, not current). Set `episode_length = min(episode_length, len(self.df) - 2)` for bounds safety. Matches `run_episode_v2` which reads `data.iloc[i+1]`.

*3.5.2 (F3) Hard MDD at -10%:* New constant `HARD_MDD_LIMIT = -0.10`. Episode terminates with `MDD_TERMINAL_PENALTY = -5.0` when `current_dd <= HARD_MDD_LIMIT`. `risk_tolerance` retains `tolerance_norm` for de-risking aggressiveness conditioning only, not the breach threshold.

*3.5.3 (F1+F5) Differential Sharpe Ratio:* Per-step reward replaced with DSR (Moody & Saffell 1998).
- `R_t = portfolio_ret - rf_daily` (excess return, `rf_daily = 0.07/252`)
- `A_t = A_{t-1} + eta*(R_t - A_{t-1})` (EMA of excess returns)
- `B_t = B_{t-1} + eta*(R_t^2 - B_{t-1})` (EMA of squared excess returns)
- `var_t = B_{t-1} - A_{t-1}^2`
- `DSR_t = (B_{t-1} * (R_t - A_{t-1}) - 0.5 * A_{t-1} * (R_t^2 - B_{t-1})) / var_t^{3/2}` when `var_t > DSR_EPS` else `0.0`
- Constants: `ETA = 0.004`, `DSR_EPS = 1e-12`
- Observation extended by 2 features: `running_sharpe_A` and `running_sharpe_B` → 13/14 total.

*3.5.4 (F4) Temporal split:* V2 branch in `ml_dispatch.py` uses `temporal_split(df)` → 70/15/15 train/val/test. Select on val, report on test. `train_best_of_n_v2` receives `test_df`.

*3.5.5 Drop patch-stack:* Remove `LAMBDA_BREACH`, `LAMBDA_CASH_BY_TOL`, `LAMBDA_OUTCOME`, `OUTCOME_HORIZON_DAYS`, `LAMBDA_DOWNSIDE` and their reward terms. Remove `outcome_match_sign` import. Keep `HEDGE_DAILY_FLOOR`, `HEDGE_COST_PER_DAY`, `RISK_TOLERANCE_MDD`.

*3.5.6 Signal sanity:* Spearman rank correlation of features vs 1-5d forward returns on train split. Gate: |rho| > 0.02, p < 0.05 for at least one feature. FAIL → ship static baseline.

*3.5.7 Retrain + eval:* best_of_n_v2 on train, select on val Sharpe, evaluate on test. Gate: Sharpe > 1.0 AND MDD > -10% on test, >= static baseline.

**Design decisions:**
1. DSR (dense per-step) over terminal-episodic Sharpe — DQN needs per-step signal for 252-step episodes.
2. Hard episode termination at -10% MDD over soft penalty — guarantees constraint-feasible replay buffer.
