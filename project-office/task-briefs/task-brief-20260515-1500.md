## [Architect] Design

**Feature summary:** This feature closes the loop between the /enhance orchestrator and the Agent Builds database by writing actual token counts and per-agent run data directly to agent_build_runs and agent_build_agents tables via a standalone helper script, enabling the Agent Builds page to display estimated vs. actual token counts side by side without any new API endpoints.

**API contract:** No new endpoints. Write path is a standalone CLI script that imports ORM models directly and commits via its own SQLAlchemy session — same pattern as seed_agent_builds.py.

| `riia-ai-org/agent-ops/aggregate_metrics.py` | No change required — existing JSON-read logic handles updated actual_tokens field |

**Repository write interface:**
- `upsert_run(run_data: dict)` — upserts AgentBuildRunModel by run_id; sets recorded_at on insert only; commits
- `upsert_agents(run_id: str, agents: list[dict], actual_tokens_total: int | None)` — upserts one AgentBuildAgentModel per agent using agent_id=f"{run_id}-{role}"; maps actual_tokens dict to integer column; commits once

**Helper script CLI:** `python riia-ai-org/agent-ops/write_run_to_db.py <run_json_path> [--actual-tokens <integer>]`
- Validates required fields (run_id, app, overall_status) before upsert
- If --actual-tokens provided: injects {"total_tokens": N} into each agent's actual_tokens field in the JSON and rewrites the file (keeps aggregate_metrics.py consistent)
- Catches OperationalError (missing table) with clear message pointing to alembic upgrade head
- Exits 0 on success, 1 on error

**RF resolutions:**
- RF-4: Helper script writes actual_tokens back to JSON file so aggregate_metrics.py (which reads JSON, not DB) stays consistent without code changes.
- RF-5: DB column stays Integer (actual_tokens_total). AgentOut schema actual_tokens: Optional[dict] is populated by Experience tier read path when serving frontend. No schema file change needed.

**Edge cases:**
- EC-1: Duplicate run_id — upsert (update if exists, insert if not); no PK violation
- EC-2: Missing --actual-tokens — write actual_tokens_total as None; aggregate_metrics.py fallback handles it
- EC-3: DB file not found / tables absent — catch OperationalError, print clear message referencing alembic upgrade head, exit 1
- EC-4: Malformed JSON (missing run_id, app, overall_status) — validate before upsert, exit 1 with field name in error
- EC-5: actual_tokens dict vs integer type mismatch in JSON — handle both; extract ["total_tokens"] if dict, use directly if integer; CLI --actual-tokens always takes precedence
