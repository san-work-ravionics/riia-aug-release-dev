## [Architect] Design

**Feature summary:** The AI Commentary feature adds auto-generated, deterministic narrative text to two RITA dashboard pages: the Market Signals (Overview) section receives a cross-instrument geographic and timeframe-aware narrative on tab load, and the Strategy section receives a plain-English rationale paragraph — displayed via typewriter animation before the result grid renders — when the user clicks "Design Strategy". Both are powered by a local rule-based reasoning layer with a single `_build_narrative()` function as the designated LLM swap point.

**API contract:**
| Field | Value |
|---|---|
| Method | POST |
| Path | `/api/v1/commentary` |
| Query params | none |
| Request body | `app: str` (required), `page: str` (required), `instrument: str\|None` (required for strategy, ignored for overview) |
| Response shape | `app: str`, `page: str`, `commentary: str`, `instruments_analyzed: list[str]`, `latency_ms: float` |
| Auth required | No — matches chat_router pattern |
| HTTP 400 | Unknown app+page combination — `{"detail": "No handler registered for app='X' page='Y'"}` |
| HTTP 200 fallback | If any instrument CSV missing, fallback sentence in `commentary`; never HTTP 500 |

**Reasoning layer design notes (for Engineer):**
- Dispatch table: module-level dict `_DISPATCH: dict[tuple[str,str], Callable]` — `("rita","overview") -> _handle_overview`, `("rita","strategy") -> _handle_strategy`. HTTP 400 if key not found.
- `_get_df()` imported from `rita.api.v1.workflow.chat` — reuses existing mtime-based cache.
- Overview: iterates NVIDIA, ASML, NIFTY, BANKNIFTY per-instrument in try/except; resamples to W and M; computes SMA-20, RSI-14, EMA-20 slope, volume avg-20 per timeframe; classifies each instrument×timeframe as STRONG/NEUTRAL/CONSOLIDATING/WEAK/RECOVERING using RSI + price-vs-SMA thresholds with volume confirmation; ranks all 4 by composite z-score; geographic buckets: US=NVIDIA, EU=ASML, India=NIFTY+BANKNIFTY.
- Strategy: calls `get_market_summary()` + `get_sentiment_score()` + `get_allocation_recommendation()` from existing core modules; builds rationale from `recommendation`, `allocation_pct`, `rationale`, `primary_constraint` fields.
- `_build_narrative(data: dict) -> str` — single LLM swap point. Today: rule-based fragment selection. Future: replace body with LLM call, same `data` dict as context.
- NaN guard: any uncomputable metric (insufficient bars after resample) defaults to NEUTRAL classification.
- Consecutive error counter: module-level `_consecutive_errors = 0`; reset on success; `log.warning` at 3.
- DB audit: write one `CommentaryLogModel` row per request via `CommentaryLogRepository(db).create(...)`. `db` injected via `Depends(get_db)`.

**Edge cases:**
1. One of 4 overview instruments has missing/corrupt CSV — per-instrument try/except; fallback classification; HTTP 200 with partial commentary; structlog warning.
2. Strategy called without `instrument` — HTTP 400: "instrument is required for page='strategy'".
3. Commentary call slower than strategy call — `Promise.allSettled` ensures strategy grid always renders regardless of commentary timing.
4. Endpoint returns HTTP 400 (bad app+page) — JS catch shows `—` in narrator box; strategy grid still renders.
5. Resampled W/M DataFrame has fewer than 20 rows — NaN guard; defaults to NEUTRAL.
6. Multiple rapid clicks on Design Strategy — `_twToken` counter cancels in-flight typewriter from previous click.
7. Very short daily DataFrame (< 30 rows) — RSI-14 returns NaN; NaN guard applies; structlog info logs thin data warning.
