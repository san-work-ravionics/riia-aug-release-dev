## [Architect] Design

**Feature summary:** Add a persistent user portfolio store so that each authenticated user can save and retrieve a named holdings allocation (list of instrument_id + allocation_pct pairs). Phase 1 is backend-only: ORM models, repos, service, migration.

**API contract:**
| Field | Value |
|---|---|
| Method | POST |
| Path | `/api/experience/portfolio/store` (Phase 2) |
| Query params | none |
| Request body | `{ name?: str, holdings: [{instrument_id: str, allocation_pct: float}] }` |
| Response shape | `{ portfolio_id, key_id, name, holdings, created_at, updated_at, is_active }` |
| Auth required | yes — JWT |
