## Request
In rita app, we have concepts page where we try to explain the flow of the app. Let us improve the page with additional content. first add a section highliting how financial firms do investments. In a professionally run investment firm, highly sophisticated quantitative finance tools and Risk / Reward processes are used by smart professionals. They use these to evaluate various investment opportunities, calculate the Risk / Reward and as appropriate, hedge them with appropriate derivatives product(s). For the purpose of this project, we will look at a specific workflow, as shown in the diagram below, which depicts a typical set-up for such an investment firm.

Workflow steps (Step / Purpose / Scope):
- Initiation / Financial Goal / To define a clear, quantitative financial goal for a time period
- Research / Research Analyst / Focus at macro-economic trends, Goods flow and Demand / Supply to arrive at growth projections
- Research / Sentiment Analyst / Look at present market conditions, public news and investor sentiments
- Research / Technical Analyst / Use of Technical Indicators such as Candlestick patterns, charts, MACD, RSI, Stochastic indicators
- Design / Strategy Analyst / Decide on the strategy approach such as Value Investing (Buy cheap), Momentum Investing (Trade on strong upside / downside), Long Term Horizon (3 to 5 years)
- Evaluation / Scenario Analyst / Evaluate the portfolio performance through various scenarios over a short-term horizon. Any expectations of maximum drawdown (MDD) on the portfolio can trigger derivatives trade
- Execution / Execution Analyst / Take the appropriate investment and balance it with right derivatives trade
- Feedback / Outcome Analyst / Build a Closed Loop system such that the result from each trade (profit / loss) is fed back to the initial phases. This then can be used to fine-tune the investment approach

As shown above, there are a few activities, and a great deal of data processing is required before taking any trades. But such knowledge and especially data processing expertise is not available with a retail trader. Unhedged exposure during events that goes against trader views can lead to a significant impact to the portfolio value. They are thus at a very significant disadvantage. This can be attempted to solve using data Science approach and Agentic AI framework.
By applying data science methods, the time-series market data can be transformed into actionable intelligence, achieving both stable performance and risk control.
With Agentic AI, a certain Risk/Reward policy can be set against which AI agents can validate the trades / portfolio and provide a Risk Informed view.
The above two elements will become the core of the proposed Risk Informed Investment Approach (RIIA). With rich time-series data availability and applying data science concepts, machine learning models can be built to do several key tasks, namely:
- Interpreting market volatility conditions using hybrid time-series and ML techniques.
- Modelling risk exposures through dynamic simulations.
- Identifying flow patterns that limits excessive leverage buildup.
- Optimizing trade sizing and execution under capital and margin constraints.

Then use the tab structure to call out each agent relevant to the step and then link appropriate plots to explain how the agent works. Follow the CRISP-DM page in Data science app on how to use the tab structure. Create a feature folder for this and add requirements md and plan md file.

1. **Empty / missing API response per tab** — Each endpoint wrapped in `Promise.allSettled`; rejected/empty result renders `_noData(canvasId)` placeholder instead of throwing. One tab's failure must not blank the others.
2. **Null / absent fields in rows** — `market-signals` rows may have null `rsi_14`/`macd`/`ema_*`; `performance.*` keys vary (`sharpe_ratio` vs `sharpe`). Use defensive `parseFloat(... ?? fallback) || null` coalescing; map nulls to `null` (not `0`) so Chart.js gaps render (`spanGaps:false`).
3. **Chart render failure / missing canvas** — `mkChart` targets existing canvas id; if a panel canvas is absent the render helper no-ops rather than crashes. Wrap whole agent-block in try/catch so a render error never breaks the Concepts page.
4. **Backend offline / all endpoints reject** — `aw-status` shows nothing fatal; all eight canvases fall back to `_noData`; existing Cards 1–6 continue functioning.
