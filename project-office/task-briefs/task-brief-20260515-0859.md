## [Architect] Design

**Feature summary:** This change closes four live defects on the Agent Builds page (P1 trend lines always flat, P1 skill version history missing improvement data, P2 token estimate result cards never populate, P3 recent_commits rendered as [object Object]) and adds Actual Token Tracking: the ability to capture real Claude API token usage per agent, store it in the run JSON and database, display actual versus estimated consumption in the UI, and feed real data back into the forecast calibration engine in aggregate_metrics.py. Together these changes make the Agent Builds page a reliable, data-accurate monitoring tool for pipeline quality and cost.

1. metrics.json missing or malformed: skill_version_history falls back to DB-derived stubs (skill_file only, all other fields None) — no exception raised
