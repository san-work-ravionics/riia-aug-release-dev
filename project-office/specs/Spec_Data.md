# RITA — Data Specification

High-density reference for AI agents. Read before any code that reads, writes, or seeds data files.

## 1. Directory Layout

```
riia-jun-release/
└── data/
    ├── raw/          ← SOURCE DATA — read-only, never modify or delete
    │   ├── NIFTY/
    │   ├── BANKNIFTY/
    │   ├── ASML/
    │   ├── NVIDIA/
    │   ├── ATHER/        ← Ather Energy (yf: ATHER.NS) — added 2026-05-20 via seed_new_instruments.py
    │   ├── RELIANCE/     ← Reliance Industries (yf: RELIANCE.NS) — added 2026-05-20
    │   ├── SBIN/         ← State Bank of India (yf: SBIN.NS) — added 2026-05-20
    │   ├── ASRNL/        ← ASR Nederland (yf: ASRNL.AS) — added 2026-05-20
    │   ├── ATO/          ← Atos SE (yf: ATO.PA) — added 2026-05-20
    │   ├── AEX/          ← AEX Index (yf: ^AEX) — added 2026-05-20
    │   ├── DJI/          ← Dow Jones Industrial Average (yf: ^DJI) — added 2026-05-20
    │   └── IXIC/         ← Nasdaq Composite (yf: ^IXIC) — added 2026-05-20
    ├── input/        ← PROCESSED / DAILY-UPDATED — read-only for the API
    │   ├── DAILY-DATA/   ← daily FnO records + recent manual OHLCV
    │   ├── NIFTY/        ← empty (.gitkeep) — future processed output
    │   ├── BANKNIFTY/    ← empty (.gitkeep)
    │   ├── ASML/
    │   ├── NVIDIA/
    │   ├── ATHER/        ← normalized output from seed_new_instruments.py
    │   ├── RELIANCE/     ← normalized output from seed_new_instruments.py
    │   ├── SBIN/         ← normalized output from seed_new_instruments.py
    │   ├── ASRNL/        ← normalized output from seed_new_instruments.py
    │   ├── ATO/          ← normalized output from seed_new_instruments.py
    │   ├── AEX/          ← normalized output from seed_new_instruments.py
    │   ├── DJI/          ← normalized output from seed_new_instruments.py
    │   └── IXIC/         ← normalized output from seed_new_instruments.py
    └── output/       ← WRITTEN BY THE API — model artefacts, backtest results
        ├── NIFTY/
        └── BANKNIFTY/
```

**Rules:**
- `data/raw/` — written only by the data refresh service (`services/data_refresh.py`). Previously read-only; Feature 16 adds automated writes for companion `_yf.csv` files (NIFTY/BANKNIFTY) and incremental `_daily.csv` appends for all other instruments.
- `data/input/` — rebuilt automatically by `rebuild_input()` in the data refresh service. For NIFTY and BANKNIFTY, the input CSV is re-generated by merging the primary source CSV with the companion `_yf.csv`. For all others, the `_daily.csv` is updated directly.
- `data/output/` — the only directory the API writes to for model artefacts (model `.zip` files, backtest artefacts).

**Adding New Daily Data (Feature 16):** Use the data refresh command: `POST /api/v1/instrument/refresh-all` or the `/refresh-all-instruments-data` slash command. For NIFTY and BANKNIFTY, data is written to companion `_yf.csv` files (`data/raw/NIFTY/nifty_yf.csv`, `data/raw/BANKNIFTY/banknifty_yf.csv`). For all other instruments, rows are appended to the existing `_daily.csv`. The old `nifty_manual.csv` / `banknifty_manual.csv` workflow is superseded (files retained but not required).

## 4. `load_ohlcv_csv()` — The Canonical Data Loader

**Location:** `src/rita/core/data_loader.py`

All code that reads OHLCV CSVs must use this function. Do not write ad-hoc `pd.read_csv()` calls in routers or services.

**Renamed in Feature 09:** `load_nifty_csv` → `load_ohlcv_csv` (instrument-agnostic name).
A backward-compat alias `load_nifty_csv = load_ohlcv_csv` is retained in data_loader.py so older imports continue to work.

```python
from rita.core.data_loader import load_ohlcv_csv

df = load_ohlcv_csv("data/raw/NIFTY/merged.csv")
# Returns: DataFrame with DatetimeIndex, columns = Open, High, Low, Close, Volume
# Index is timezone-naive (converted from IST if needed)
# Sorted ascending by date
```

**What it handles automatically:**
- Lowercase → Title Case column renaming (`close` → `Close`, `shares traded` → `Volume`)
- Timezone-aware IST dates (`+05:30`) → tz-naive
- Plain ISO dates (`2001-03-09`) → datetime
- `dd-MMM-yyyy` format (`16-MAR-2026`) → datetime
- Sorts by date ascending

**Output columns:** `Open`, `High`, `Low`, `Close`, `Volume` (Volume may be NaN if the source file has no volume column)

### `find_instrument_csv()` — Dynamic Glob-Based Resolution

**Location:** `src/rita/core/data_understanding.py`

**Updated in Feature 09** from a hardcoded per-instrument mapping to a dynamic glob:

```
NIFTY:     data/raw/NIFTY/merged.csv              (exception — explicit path)
BANKNIFTY: data/raw/BANKNIFTY/banknifty_daily_25yr_rounded.csv (exception — explicit path)
All others:
  Priority 1: first CSV in data/raw/{TICKER}/
  Priority 2: first CSV in data/input/{TICKER}/   (fallback)
```

New instruments onboarded via `POST /api/v1/instrument/onboard` are automatically discovered by this logic once their CSV is written to `data/raw/{TICKER}/`.

## 5. DB Seeding — What Goes In and Why

All 12 instruments are seeded into `market_data_cache` at startup (2025+2026 window). Seeding changed from "only when table empty" to **upsert-on-startup** — any new entry in `_SEED_INSTRUMENTS` is added automatically on next container restart.

**Seeding window: 2025 + 2026 only**

| Instrument | Source file | Region | Year filter |
|---|---|---|---|
| NIFTY | `data/raw/NIFTY/merged.csv` + `data/input/DAILY-DATA/nifty_manual.csv` | India | 2025 + 2026 |
| BANKNIFTY | `data/raw/BANKNIFTY/banknifty_daily_25yr_rounded.csv` | India | 2025 + 2026 |
| RELIANCE | `data/raw/RELIANCE/reliance_daily.csv` | India | 2025 + 2026 |
| SBIN | `data/raw/SBIN/sbin_daily.csv` | India | 2025 + 2026 |
| ASML | `data/raw/ASML/asml_2001-2026.csv` | EU | 2025 + 2026 |
| ASRNL | `data/raw/ASRNL/asrnl_daily.csv` | EU | 2025 + 2026 |
| ATO | `data/raw/ATO/ato_daily.csv` | EU | 2025 + 2026 |
| AEX | `data/raw/AEX/aex_daily.csv` | EU | 2025 + 2026 |
| NVIDIA | `data/raw/NVIDIA/nvda_daily_25yr_rounded.csv` | US | 2025 + 2026 |
| DJI | `data/raw/DJI/dji_daily.csv` | US | 2025 + 2026 |
| IXIC | `data/raw/IXIC/ixic_daily.csv` | US | 2025 + 2026 |
| ATHER | `data/raw/ATHER/ather_daily.csv` | India | 2025 + 2026 — **not yet available** (IPO 2025, yfinance not indexed) |

Each instrument is skipped if already present in `market_data_cache`. Uses **single `db.add_all()` + one commit** per instrument — completes in < 2 seconds each.

**Why not the full 25-year history?**
Seeding 6,594 rows row-by-row (one `upsert()` = one `commit()`) took 78 seconds at startup. The bulk insert is < 2 seconds. 260–300 rows is sufficient for all technical indicators:

| Indicator | Bars needed | Available |
|---|---|---|
| RSI-14 | 14 | ~266 |
| MACD (12, 26, 9) | 26 | ~266 |
| Bollinger Bands (20, 2σ) | 20 | ~266 |
| EMA-50 | 50 | ~266 |
| ATR-14 | 14 | ~266 |

**Adding new year data:** When 2026 ends, update the seed filter in `main.py` to `year >= 2025` → `year >= 2026`, and move 2026 data from `nifty_manual.csv` into `merged.csv`.

## 6. Adding New Daily Data (Ongoing Process)

As of Feature 16, daily data refresh is automated. Use `POST /api/v1/instrument/refresh-all` or the `/refresh-all-instruments-data` slash command. The service fetches delta rows from yfinance for all 11 instruments (ATHER skipped), rebuilds input CSVs, and upserts new rows into market_data_cache.

The legacy `nifty_manual.csv` / `banknifty_manual.csv` workflow is superseded — these files are retained and still merged by `load_instrument_data()` as a supplement, but are no longer the primary mechanism for live data. New rows written manually to these files continue to be picked up automatically on the next load.

## 7. Model Output Files

Trained model artefacts are written to `data/output/{INSTRUMENT}/`:

```
data/output/NIFTY/
    {model_version}.zip       ← stable-baselines3 DoubleDQN model
    backtest_results.csv      ← daily portfolio values from latest backtest
    performance_summary.json  ← aggregated KPIs
```

**These files survive DB deletion.** The DB tracks run metadata (timestamps, Sharpe, MDD) but the model weights and backtest CSVs are on disk. If the DB is lost, the models can still be loaded and used — only the run history is lost.

## 9. AI Agent Directives

1. **Never write to `data/raw/`** — it is the immutable source of truth.
2. **Never call `pd.read_csv()` directly on OHLCV files** — always use `load_ohlcv_csv()`. It handles all date formats and column normalisation.
3. **Do not seed the full 25-year history** — the 2025+2026 window is the correct design. Full history causes slow startup.
4. **To add a new instrument to the DB seed:** add an entry to `_SEED_INSTRUMENTS` in `main.py` and commit its CSV files to `data/raw/{TICKER}/` and `data/input/{TICKER}/`. The upsert-on-startup logic will insert it on next container restart; the deploy pipeline rsyncs the CSVs to EC2 automatically.
5. **Automated refresh via `POST /api/v1/instrument/refresh-all` is the primary live feed.** `nifty_manual.csv` is a manual supplement — it is still merged by `load_instrument_data()` and remains valid for day-level backfill, but the data_refresh service supersedes it for routine daily updates.
6. **New instruments auto-onboard via API** — use `POST /api/v1/instrument/onboard`. The service writes to `data/raw/{TICKER}/` and `data/input/{TICKER}/` automatically; `find_instrument_csv()` discovers them via glob.

## 10. New Instrument Data Layout (Feature 09)

New instruments onboarded via `POST /api/v1/instrument/onboard` follow this layout:

```
data/
├── raw/
│   └── {TICKER}/
│       └── {ticker_lower}_daily.csv    ← yfinance OHLCV download (never modified after write)
└── input/
    └── {TICKER}/
        └── {ticker_lower}_daily.csv    ← normalized: Open/High/Low/Close/Volume
                                           tz-naive, sorted asc, year >= 2010
```

| Window | Purpose | Filter |
|---|---|---|
| 2009-09-01 → 2009-12-31 | Indicator warmup (not in input CSV) | Fetched, not saved |
| 2010-01-01 → present | Full training history | `data/input/{TICKER}/` |
| 2025-01-01 → present | Runtime DB cache | `market_data_cache` |

**Onboarding pipeline** (Feature 09 — `src/rita/services/instrument_onboard.py`):
1. `fetch_raw_data(ticker)` → downloads from yfinance, writes `data/raw/{TICKER}/{ticker_lower}_daily.csv`, raises ValueError if < 100 rows
2. `process_to_input(ticker, raw_path)` → normalizes, filters year >= 2010, writes `data/input/{TICKER}/{ticker_lower}_daily.csv`
3. `seed_market_cache(db, ticker, currency)` → reads input CSV, filters year >= 2025, bulk inserts into `market_data_cache`

The four original instruments (NIFTY, BANKNIFTY, ASML, NVIDIA) are seeded at startup in `main.py` lifespan and are NOT affected by the new onboarding flow.
